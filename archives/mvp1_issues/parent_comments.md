## Background
Enable users to comment on surf sessions, creating richer social engagement. Comments should be lightweight, easy to use, and follow the existing interaction patterns (similar to shakas).

## Overview
This parent issue tracks the complete Comments System implementation. See sub-issues for specific backend and frontend tasks.

## Sub-Issues
- Backend: Comments table and API endpoints
- Frontend: CommentModal component
- Frontend: Add comment UI to tiles and detail view

## Key Requirements
- 500 character limit on comments
- Any logged-in user can comment on any session
- Comment count displayed on session tiles and detail views
- No edit/delete functionality in MVP_1

## Acceptance Criteria
- [ ] All sub-issues completed and closed
- [ ] Users can view and post comments on any session
- [ ] Comment counts display accurately across all views
- [ ] No critical bugs in comment functionality
